3. How We Use the Data

  • Authentication: account data is used exclusively to identify the logged-in user.
  • Push notifications: the FCM token is used to deliver match start, goals, cards, and end-of-game alerts.
  • Advertising: AdMob uses the advertising identifier to display relevant ads. It can be disabled in Android settings.
  • Analytics: aggregated and anonymized data to identify failures and improve the app.
  • Local data: favorite teams, search history, and preferences remain on the device to personalize the experience.

⚠️ Match Time Pro does not sell, rent, or share personal data with third parties for their own commercial purposes.

API-Football (API Sports)

Provides match data. Requests pass through our intermediary server, which only sends query parameters (team, league, season IDs). No personal user data is transmitted.
api-football.com

GNews API

Provides football news. Requests pass through our intermediary server, which only sends search terms. No personal user data is transmitted.
gnews.io/privacy-policy

Intermediary server (backend)

Match Time Pro uses its own server that caches sports data, avoiding redundant calls to external APIs. The server stores only the FCM token (for sending notifications) and public match data. It does not store personally identifiable information.

5. Data Retention and Deletion

  • Firebase account: kept while active. Users may delete the account in Settings → Account → Delete account.
  • FCM token: periodically renewed by the system, with no link to identifiable data. Invalid tokens are automatically removed from the server.
  • Local data: erased upon app uninstallation.
  • AdMob and Analytics: managed by Google according to their retention policies.
